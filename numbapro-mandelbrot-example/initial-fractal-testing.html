def mandel(x, y, max_iters):
  """
    Given the real and imaginary parts of a complex number,
    determine if it is a candidate for membership in the Mandelbrot
    set given a fixed number of iterations.
  """
  c = complex(x, y)
  z = 0.0j
  for i in range(max_iters):
    z = z*z + c
    if (z.real*z.real + z.imag*z.imag) >= 4:
      return i

  return max_iters

Using Numba (In my own words)¶

  • "Numpy-aware" dynamic compiler
  • Optimized compiling for Python
    • Compiles Python syntax to more readable machine code.
  • NumPy arrays are seen as an object that is assigned a region in memory.
    • A region is a collection of objects that can stored and removed from memory efficiently (similar to a stack).
    • However, the objects (NumPy arrays) are able to live longer than objects in "the stack."
    • The objects are stored in a connected range of memory.

The speed up is seen because of the way the NumPy arrays are typed in memory regions.

Use @jit (@autojit is dep.) before functions that should be compiled with Numba. The functions are compiled to machine code that can be read by the CPU if ("if it can resolve the types used.") In this case Numba is aware of the type NumPy arrays explored above.

  • autojit is depreciated in python 3, use jit instead

Explanation in my own words¶

The types used in NumPy are types that can be converted to machine code just like and int can be converted to machine code in C. The CPU can read machine code much faster.

Convert the dynamically typed, interpreted Python code into statically typed, compiled machine code.
